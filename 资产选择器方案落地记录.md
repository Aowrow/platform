# 资产选择器方案落地记录

这次针对“参考图选择混乱、图片多了难找、分镜图分割更需要选输出图”的问题，落地了统一资产选择器方案。

目标是：

- 不再在表单里直接平铺所有参考图
- 不同功能使用不同默认筛选规则
- 让分镜图分割默认更容易找到“分镜图生成”的输出图

---

## 1. 后端：资产筛选接口增强

更新文件：

- `backend/src/modules/assets/assets.service.ts`
- `backend/src/modules/assets/assets.controller.ts`

现在 `GET /api/assets` 支持按这些条件筛选：

- `assetType`
- `mediaType`
- `bizType`
- `taskId`
- `keyword`

示例：

- 只看上传图：`/api/assets?assetType=input&mediaType=image`
- 只看分镜图生成输出：`/api/assets?assetType=output&mediaType=image&bizType=fenjingtu`

这为前端资产选择器提供了真正可用的筛选能力，而不是每次全量拉数据再本地硬筛。

---

## 2. 前端：统一资产选择器弹层

更新文件：`frontend/components/feature-form.tsx`

这次把原来的“表单里直接平铺全部参考图”替换成了统一的资产选择器交互：

- 表单区只显示当前已选图片
- 点击“选择图片”打开弹层
- 弹层里可以：
  - 按功能默认规则筛选
  - 搜索文件名或任务编号
  - 点击缩略图选择

这样即使图片越来越多，选择体验也仍然可控。

---

## 3. 不同功能的默认策略

### 3.1 分镜图生成 `fenjingtu`

默认策略：

- `mode = multiple`
- 默认筛选：只看 `input` 图片

也就是说：

- 更适合从上传图里选择一张或多张参考图

### 3.2 分镜图分割 `fenjingtu-splitter`

默认策略：

- `mode = single`
- 默认筛选：只看 `fenjingtu` 的 `output` 图片

同时还提供切换：

- 只看分镜图生成结果（推荐）
- 看全部输出图
- 看全部图片

这就解决了你提出的核心问题：

- 分镜图分割从逻辑上更适合选“分镜图生成”的输出图
- 而不是一堆上传图混在一起

---

## 4. 分镜图分割改为单选输入

因为 `fenjingtuSplitter.json` 当前只使用一张图，所以这次前端也同步做了交互约束：

- `fenjingtu-splitter` 的参考图选择是单选
- 选一张图后会直接替换之前选择

这样前端行为和工作流能力一致，不会再出现“看起来可以多选，但实际只用第一张”的误导。

---

## 5. 体验上的直接提升

改完之后，现在参考图选择逻辑更符合真实业务：

- 分镜图生成：默认挑上传图
- 分镜图分割：默认挑分镜图生成结果

而且选择器支持：

- 搜索
- 筛选
- 单选 / 多选区分

这会比之前图片一多就只能靠手动翻找好很多。

---

本次改动后已验证：

- `backend` build 通过
- `frontend` build 通过
