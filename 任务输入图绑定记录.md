# 任务输入图绑定记录

这次改动的目标是：

- 不只是让上传图片变成“可选参考图”
- 而是让这些图片在数据库里真正和某次任务建立关联
- 这样后续就能清晰区分：
  - 哪些图是当前任务的输入图
  - 哪些图是当前任务的输出图

---

## 1. 后端：创建任务时回写输入资产关联

更新文件：`backend/src/modules/tasks/tasks.service.ts`

本次在 `createForFeature()` 中新增逻辑：

- 从 `inputParams.referenceImageIds` 读取当前选中的参考图资产 ID
- 在任务创建成功后，执行 `assets.updateMany()`
- 把这些输入图片的 `taskId` 回写为当前任务 ID

更新条件做了约束：

- 只更新当前用户的图片
- 只更新 `assetType = input` 的资产

这样可以保证：

- 你在前端勾选的参考图，不只是临时用于生成 URL
- 也会在数据库层面真正成为这个任务的“输入图资产”

---

## 2. 前端：任务面板区分输入图和输出图

更新文件：`frontend/components/feature-form.tsx`

之前任务调试区里只统一展示任务关联图片，现在改成明确分为两块：

- 输入图片
- 输出图片

实现方式：

- 根据 `taskAssets[taskId]` 里的 `assetType` 区分
- `assetType = input` 归为输入图片
- `assetType = output` 归为输出图片

这样你现在在“分镜图”页里能更直观地看见：

- 这次任务到底用了哪些参考图
- 这次任务最终生成了哪些结果图

---

## 3. 完成后的效果

现在任务和图片的关系已经更清晰：

- 上传图片 -> 进入资产管理
- 选中图片提交任务 -> 数据库中这些图片会绑定到当前任务
- 任务成功生成 -> 输出图也会绑定到同一个任务

因此同一个任务现在已经具备：

- 输入图资产
- 输出图资产

这对后续做任务详情页、输入输出对比、结果复盘都非常重要。

## 5. 补充修复：任务完成后输出图自动刷新

后来又发现一个体验问题：

- 任务完成后，输出图片区域不会立刻自动出现
- 必须手动刷新页面才能看到

问题原因是：

- 前端虽然会自动刷新任务状态
- 但“任务资产列表”只在切换任务时重新加载
- 当同一条任务从 `running` 变成 `success` 时，图片资产不会自动再次请求

这次更新文件：`frontend/components/feature-form.tsx`

修复方式：

- 让同步任务资产的 `useEffect` 不只依赖 `selectedTask.id`
- 还依赖：
  - `selectedTask.status`
  - `selectedTask.progress`
  - `selectedTask.resultSummary.imageCount`

这样当任务状态变化或结果数量变化时，前端会重新拉取该任务的资产列表。

修复后的效果：

- 任务跑完后，输出图片区域会自动刷新
- 不再需要手动刷新整个页面才能看到结果图
