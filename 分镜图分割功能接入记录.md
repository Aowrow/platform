# 分镜图分割功能接入记录

这次新增了一个新的功能页：`分镜图分割`。

对应工作流文件：

- `backend/workflows/fenjingtuSplitter.json`

这个功能是一个简单的图片处理任务：

- 输入一张图片
- 输入 `rows` 和 `columns`
- 使用工作流中的 `ImageStoryboardSplitter` 节点进行分割

---

## 1. 工作流结构确认

当前工作流里的关键节点是：

- `31`：`LoadImageURL`
  - 输入字段：`url`
- `18`：`ImageStoryboardSplitter`
  - 输入字段：`rows`
  - 输入字段：`columns`
  - 输入图像来自节点 `31`

所以这次接入沿用了“平台图片 URL -> 工作流 URL 节点”的正式路线。

---

## 2. 后端接入

更新文件：`backend/src/modules/features/feature-registry.ts`

新增 feature：

- `fenjingtu-splitter`

主要配置：

- 工作流文件：`fenjingtuSplitter.json`
- 输入字段：
  - `referenceImages`
  - `rows`
  - `columns`

参数映射方式：

- 选中的第一张参考图 URL -> `31.inputs.url`
- `rows` -> `18.inputs.rows`
- `columns` -> `18.inputs.columns`

说明：

- 这个工作流目前是单图输入，所以即使前端上传了多张参考图，也只会使用第一张

---

## 3. 前端接入

更新文件：

- `frontend/lib/feature-catalog.ts`
- `frontend/components/app-shell.tsx`

改动内容：

- 在前端功能目录中新增 `fenjingtu-splitter`
- 为它配置了表单字段：
  - 分镜图图片
  - 行数
  - 列数
- 在左侧导航中增加“分镜图分割”入口

这样现在你可以直接访问：

- `/features/fenjingtu-splitter`

来测试这个新功能。

---

## 4. 输入输出路线

这个新功能继续沿用和“分镜图生成”一致的资源路线：

- 输入图：来自平台上传资产
- 输入方式：平台资源 URL
- 输出图：继续走任务结果同步到 MinIO 与 `assets`

因此不需要额外设计新的文件处理方式。

---

本次改动后已验证：

- `frontend` build 通过
